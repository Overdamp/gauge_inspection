ยินดีต้อนรับกลับสู่โหมดนักพัฒนาครับคุณลุค! หลังจากที่เคลียร์พื้นที่เครื่องจนโล่งพร้อมลุยแล้ว เรามาเจาะลึกการใช้งาน Postman แบบสเต็ปบายสเต็ปกันครับ

เพื่อให้เห็นภาพและนำไปประยุกต์ใช้กับงาน AI และ Triton Server ของคุณได้ทันที ผมจะแบ่งการใช้งานออกเป็น 4 ส่วนหลักๆ ตั้งแต่พื้นฐานไปจนถึงระดับโปรครับ

🌟 ส่วนที่ 1: จัดระเบียบการทำงาน (Workspace & Collections)
ก่อนจะเริ่มยิง API เราควรสร้างแฟ้มจัดเก็บให้เป็นระเบียบก่อนครับ จะได้ไม่หลงเวลาที่โปรเจกต์ใหญ่ขึ้น

Workspaces (พื้นที่ทำงาน): มุมซ้ายบน ให้กดเมนู Workspaces -> Create Workspace (ตั้งชื่อเช่น "Triton AI Projects") นี่คือห้องทำงานใหญ่ของคุณ

Collections (แฟ้มสะสม API): แถบด้านซ้ายมือ ให้คลิกไอคอน + เพื่อสร้าง Collection ใหม่ (ตั้งชื่อเช่น "Triton ResNet50" หรือ "ROS2 Camera Stream")

ข้อดี: คุณสามารถเซฟ Request หลายๆ ตัว (เช่น Health Check, Get Metadata, Infer Model) รวมไว้ใน Collection เดียวกันได้ เวลาเปิดเครื่องมาวันหลังก็กดใช้งานต่อได้เลย

🌟 ส่วนที่ 2: วิธีประกอบร่างคำสั่ง (The Request Builder)
นี่คือพื้นที่หลักตรงกลางหน้าจอที่คุณจะต้องใช้งานบ่อยที่สุดครับ ให้กด Add a request เข้าไปใน Collection จะเจอส่วนประกอบดังนี้:

1. เลือก Method (วิธีการส่ง)

GET: ใช้สำหรับ "ดึง" ข้อมูล (เช่น ดึงสถานะ Health Check ของ Triton ว่าพร้อมไหม)

POST: ใช้สำหรับ "ส่ง" ข้อมูลชุดใหญ่ไปประมวลผล (เช่น ส่งข้อมูลภาพหรือ Tensor Array ไปให้ AI ทำนายผล)

1. กรอก URL เป้าหมาย

ใส่ที่อยู่ของเซิร์ฟเวอร์ เช่น <http://localhost:8000/v2/models/my_model/infer>

1. จัดเตรียม Headers (ถ้าจำเป็น)

บางระบบต้องการการยืนยันตัวตน หรือต้องบอกว่าเราส่งข้อมูลประเภทไหนไป ให้มาที่แท็บ Headers

สำหรับงาน AI ทั่วไป: มักจะใช้ Key: Content-Type และ Value: application/json (แต่ปกติ Postman จะใส่ให้อัตโนมัติถ้าเราเลือก Body เป็น JSON)

1. ปั้นข้อมูล Payload (แท็บ Body) - สำคัญมากสำหรับ AI
เวลาส่งข้อมูลไปให้โมเดลทำนาย ต้องมาที่แท็บนี้ครับ

เลือก raw

คลิก Dropdown เล็กๆ ด้านขวา (ข้างๆ คำว่า Text) เปลี่ยนเป็น JSON

พิมพ์โครงสร้าง JSON ลงไป (ตัวอย่างสำหรับ Triton):

JSON
{
  "inputs": [
    {
      "name": "input_image",
      "shape": [1, 3, 224, 224],
      "datatype": "FP32",
      "data": [0.5, -0.2, 0.1, 0.9]
    }
  ]
}
🌟 ส่วนที่ 3: กดยิงคำสั่งและอ่านผลลัพธ์ (The Response)
เมื่อกรอกทุกอย่างครบ ให้กดปุ่ม Send สีฟ้าขนาดใหญ่มุมขวาบน หน้าต่างด้านล่างจะแสดงผลลัพธ์ที่ Server ตอบกลับมาครับ

จุดที่ต้องสังเกตมี 3 อย่าง:

Status Code:

200 OK = สำเร็จ ข้อมูลถูกต้อง โมเดลทำนายผลเรียบร้อย

400 Bad Request = เราส่งข้อมูลผิด (เช่น Shape ไม่ตรงกับที่โมเดลต้องการ, พิมพ์ JSON ผิดฟอร์แมต)

500 Internal Server Error = Server ล่ม หรือโมเดลใน Triton แครช

Time & Size: ดูว่า AI ใช้เวลาประมวลผลกี่มิลลิวินาที (ms) และส่งข้อมูลกลับมาขนาดกี่ KB (สำคัญมากในการ Optimize ระบบหุ่นยนต์)

Response Body: ก้อน JSON ที่ตอบกลับมา เช่น ผลลัพธ์ความน่าจะเป็น (Probabilities) หรือค่า Bounding Box

🔥 ส่วนที่ 4: ฟีเจอร์ระดับ Pro (เอาไปใช้ต่อในโค้ด)

1. เปลี่ยนตัวแปร (Environments)
สมมติว่าคุณลุคเทสบน localhost:8000 จนเสร็จ แล้วจะย้ายไปรันบนหุ่นยนต์จริงที่มี IP 192.168.1.100 คุณไม่ต้องมานั่งแก้ URL ทุกอันครับ

กดที่ไอคอนรูปตา (Environment quick look) มุมขวาบน -> กด Add

สร้างตัวแปรชื่อ {{base_url}}

ในช่อง URL ให้พิมพ์ http://{{base_url}}/v2/... แทน เวลาสลับเครื่องก็แค่สลับ Environment ได้เลย

1. ปุ่มเสกโค้ด (Code Snippet)
นี่คือฟีเจอร์ที่ประหยัดเวลาที่สุด! เมื่อคุณเทส Postman จนยิงผ่านแล้ว และต้องการเอาไปเขียนเป็น Python Node ใน ROS 2:

มองหาไอคอน </> ที่แถบเมนูด้านขวามือ

คลิกแล้วเลือกภาษา เช่น Python - Requests หรือ C++ - cURL

Postman จะ Generate โค้ดที่เขียน Request หน้าตานั้นให้เป๊ะๆ คุณก๊อปปี้ไปแปะในสคริปต์ของคุณได้เลยครับ!

1. ยิงผ่าน gRPC (สำหรับงานที่ต้องการความเร็วสูง)
Triton Server รัน gRPC บนพอร์ต 8001 ซึ่งรับส่งข้อมูล Tensor ได้เร็วกว่า HTTP มาก Postman เวอร์ชันใหม่รองรับการสร้าง New gRPC Request แล้วครับ สามารถโหลดไฟล์ .proto ของ Triton เข้าไป แล้วยิงเทสแบบ gRPC ได้จากหน้า UI เลยครับ
