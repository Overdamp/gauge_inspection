# ระบบอ่านค่ามิเตอร์เข็มอัตโนมัติ (Analog Gauge Inspection System)

## 🎯 สรุปโปรเจค

นี่คือ **ระบบอ่านค่ามิเตอร์เข็มอัตโนมัติ** ที่ใช้เทคนิค Computer Vision และ Deep Learning เพื่ออ่านค่าจากภาพมิเตอร์เข็มโดยอัตโนมัติ

ระบบนี้จะรับภาพมิเตอร์ที่ถูกครอบขาดไว้แล้ว และใช้เทคนิค advanced หลายอย่าง เช่น:
- **Image Segmentation** - แบ่งส่วนของมิเตอร์
- **OCR (Optical Character Recognition)** - อ่านตัวเลขบนหน้าปัด
- **Ellipse Fitting** - ปรับให้ตรงกับรูป Ellipse
- **Needle Detection** - ตรวจหาตำแหน่งเข็ม

สุดท้ายจะให้ผลลัพธ์เป็นค่าที่อ่านได้ พร้อมตัวเลขคุณภาพของการอ่าน (R² Score)

---

## 📋 วัตถุประสงค์ของโปรเจค

1. **ตรวจหาส่วนประกอบของมิเตอร์**: ระบุตำแหน่งของเข็ม หน้าปัด และตัวเลข
2. **แบ่งส่วนภาพ (Segmentation)**: แยกแต่ละส่วนของมิเตอร์ออกมา
3. **อ่านตัวเลข (OCR)**: ดึงตัวเลขและหน่วยวัดจากหน้าปัด
4. **ปรับรูปทรง Ellipse**: ให้ความแม่นยำในการคำนวณมุมเข็ม
5. **ตรวจหาเข็ม**: ระบุตำแหน่งและมุมของเข็มชี้
6. **คำนวณค่าที่อ่านได้**: แปลงมุมเข็มเป็นค่าจริง
7. **สร้างรายงานสำหรับตรวจสอบ**: ให้ภาพการทำงาน 8 ส่วน


## 🤖 ส่วนประกอบหลักของระบบ

### **1. Segmentation Module (segmentation.py)**
**ทำไม**: แบ่งส่วนของมิเตอร์
- **โมเดล**: YOLOv8 (รุ่นการเรียนรู้)
- **รับเข้า**: ภาพมิเตอร์
- **ให้ออกมา**: ภาพแสดงตำแหน่งเข็ม, หน้าปัด, ตัวเลข
- **ตั้งค่า**: `config.yaml` ส่วน `segmentation`

### **2. Ellipse Fitter (ellipsefit.py)**
**ทำไม**: ปรับรูปวงกลมให้ตรงกับหน้าปัด
- **รับเข้า**: ภาพหน้าปัดที่ตรวจพบ
- **ให้ออกมา**: จุดศูนย์กลาง (cx, cy), พารามิเตอร์
- **ใช้สำหรับ**: คำนวณมุมของเข็มได้แม่นยำ

### **3. OCR Module (ocr_ai.py)**
**ทำไม**: อ่านตัวเลขบนหน้าปัดมิเตอร์
- **โมเดล**: Doctr (ระบบ OCR)
- **รับเข้า**: ภาพส่วนตัวเลข
- **ให้ออกมา**: ตัวเลขและตำแหน่งของมัน
- **ตัวอย่าง**: "0", "10", "20", "30" เป็นต้น

### **4. Gauge Calculator (gauge_cal.py)**
**ทำไม**: คำนวณค่าจากมุมเข็ม
- **ขั้นตอน**:
  1. หามุมของเข็มสัมพัทธ์กับศูนย์กลาง
  2. แปลงมุมเป็นค่าจริง
  3. คำนวณ R² (คะแนนการทำงาน)
- **ให้ออกมา**: ค่ามิเตอร์, หน่วยวัด, R² score

### **5. Super-Resolution (superresolution.py)**
**ทำไม**: ปรับปรุงคุณภาพของภาพก่อนประมวลผล
- **โมเดล**: Swin2SR
- **ใช้เมื่อ**: ต้องการความแม่นยำสูงขึ้น

### **6. Debug Reporter (gauge_debug.py)**
**ทำไม**: สร้างภาพรายงาน 8 ส่วน
- **ส่วนที่ 1**: ภาพต้นฉบับ
- **ส่วนที่ 2**: ภาพแบ่งส่วน (เห็นส่วนต่างๆ ด้วยสี)
- **ส่วนที่ 3**: ภาพวงกลมที่ปรับแล้ว
- **ส่วนที่ 4**: ตัวเลขที่อ่านได้
- **ส่วนที่ 5**: ตำแหน่งเข็มที่ตรวจพบ
- **ส่วนที่ 6**: กราฟปรับเทียบ
- **ส่วนที่ 7**: วิเคราะห์รัศมี
- **ส่วนที่ 8**: ผลลัพธ์สุดท้าย

## ⚙️ ไฟล์ตั้งค่า (config.yaml)

```yaml
system:
  log_level: "INFO"              # ระดับของการบันทึก
  log_file: "logs/analog_system.log"

analog_gauge:
  device: "cuda"                 # ใช้ GPU หรือ CPU
  ocr_model_dir: "models/analog_gauge_model/ocr"
  
  segmentation:
    model_path: "models/analog_gauge_model/segmentation/best_segment_v2.pt"
    conf: 0.5                    # ค่า confidence (การมั่นใจ)
    iou: 0.5                     # ค่า IoU สำหรับการตรวจสอบ
    verbose: False               # แสดงรายละเอียด ใช่/ไม่ใช่
  
  superresolution:
    model_name: "caidas/swin2SR-lightweight-x2-64"
  
  debug:
    enabled: true                # เปิด/ปิด debug mode
    output_dir: "debug_output"   # โฟลเดอร์เก็บภาพ debug
```

## 🎓 เทคนิคขั้นสูง

1. **คำนวณมุมเข็ม**:
   - ใช้ Anti-shadow scoring เพื่อหาเข็มที่แท้จริง
   - คำนวณมุมเทียบกับจุดศูนย์กลาง
   - แปลงมุมเป็นค่ามิเตอร์

2. **ปรับเทียบ**:
   - ใช้ Linear mapping: มุม → ค่า
   - R² score บอกความแม่นยำ
   - รองรับมิเตอร์แบบซับซ้อน

3. **ภาพมาส์**:
   - แสดงด้วยสีต่างๆ (แต่ละประเภทมีสีต่างกัน)
   - ใช้สำหรับคำนวณและตรวจหา
   - มีโมเดลหลายเวอร์ชัน

4. **OCR**:
   - อ่านตัวเลข (0, 10, 20, 30)
   - ใช้ตำแหน่งตัวเลขเพื่อปรับเทียบ
   - รองรับหน่วยต่างๆ (PSI, BAR, kg/cm² ฯลฯ)
